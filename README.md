💡 Hue-CLI (v2 SNI Aligned)A lightweight, professional-grade Command Line Interface (CLI) for controlling Philips Hue lights and rooms. Built with Node.js and undici, this tool is specifically designed to handle the Hue V2 API security requirements, including self-signed certificates and Server Name Indication (SNI) validation.🔍 Bridge Discovery (Future Feature)Note: Automatic bridge discovery via N-UPnP or mDNS is currently in development.For now, please obtain your Bridge IP and Application Key manually. You can find your Bridge IP via your router's DHCP list or the official Hue App (Settings > Hue Bridges > (i)). To generate an appKey, refer to the official Hue Developer Guide.🛠 PrerequisitesBefore installing, ensure your system meets the following requirements:Node.js: version 18.0.0 or higher (Required for fetch and undici support).NPM: Normally bundled with Node.js.Philips Hue Bridge: Square version (V2) with the latest firmware.CA Certificate: You must have your bridge's huebridge_cacert.root.pem.📦 DependenciesThe project relies on the following high-performance libraries:undici: For HTTP/1.1 and HTTP/2 support with custom TLS agents.commander: For a robust Unix-like command-line interface.ini: For managing cross-platform configuration files.readline-sync: For interactive setup during configuration.🚀 Installation1. Clone and InstallBashgit clone https://github.com/yourusername/hue-cli.git
cd hue-cli
npm install
2. Make it GlobalTo run the hue command from any terminal (Kitty, PowerShell, Bash, etc.), link the package to your system:Linux / macOS:Bashsudo npm link
Windows (Run PowerShell as Administrator):PowerShellnpm link
⚙️ ConfigurationThe configuration is stored globally in your home directory (~/.hue-config.ini) to ensure the tool works regardless of which folder your terminal is currently in.Initialize Settings:Bashhue configure
You will be prompted for:Bridge IP: (e.g., 192.168.1.50)Bridge ID: (The unique ID, e.g., ECB5FA...)App Key: Your generated whitelist username.Cert Path: Absolute path to your huebridge_cacert.root.pem.Generate Aliases:Automatically scan your bridge and create "friendly names" (aliases) for all your lights and rooms:Bashhue map
📖 Usage GuideDisplay AliasesIf you forget the name of a light or room, use the show command to see a formatted table of your mapped resources:Bashhue show
Control CommandsCommandDescriptionExampletoggleTurn a resource on or offhue toggle kitchen_lights ondimSet brightness (0-100)hue dim office_desk 75blinkFlash a light to identify ithue blink light_3listView raw API datahue list roomGeneral Optionshue --help: Show help for all commands.hue [command] --help: Show help for a specific command.hue --version: Display the current version.🐧 Platform SpecificsLinux (Kitty/Bash/Zsh)If the hue command is not found after running npm link, ensure your npm bin path is in your $PATH. Add this to your .bashrc or .zshrc:Bashexport PATH=$PATH:$(npm get prefix)/bin
WindowsIf you receive a "script execution" error in PowerShell, you may need to set your execution policy:PowerShellSet-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
